## Task: Media Quality Debt Fix (PR #29)

Fixed 6 bugs and added comprehensive tests for `internal/media` and `internal/mediachaos` packages. +2,267 lines across 5 files.

### Bugs Fixed

| # | Bug | Fix |
|---|-----|-----|
| 1 | ETag conditional request check happened AFTER media chaos — chaos corrupted data before If-None-Match could short-circuit | Moved ETag computation + If-None-Match check BEFORE chaos `Apply()` |
| 2 | CDN headers applied twice in media response path (once in ServeHTTP, again in serveMedia) | Removed duplicate `ApplyHeaders` call in `serveMedia()` |
| 3 | Media path detection used `strings.Contains` — `/api/v1/images/endpoint` falsely matched as media | Changed to `strings.HasPrefix` for extension directory paths |
| 4 | HLS/DASH content-types missing from `applyFormatCorruption()` switch — fell through to generic corruption | Added `application/vnd.apple.mpegurl` and `application/dash+xml` cases + `.m3u8`/`.mpd` extension fallbacks |
| 5 | Non-deterministic RNG in mediachaos — `rand.Int63()` seed meant different corruption per request to same path | SHA-256 seed from request path for reproducible corruption |
| 6 | `corruptHLS()`/`corruptDASH()` panic on empty input — variant 0 slices `[:1]` on empty string | Added `len(data) == 0` early return guard |
