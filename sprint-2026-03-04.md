## Task 1 ‚Äî Budget-Draining Traps Module
**Branch**: `feature/budget-draining-traps`
**Status**: COMPLETE ‚Äî PR #41 merged

### Implementation Plan (from codebase research)

**New package**: `internal/budgettrap/` ‚Äî engine with per-client escalation tracking
**Integration point**: `server/handler.go:dispatch()` ‚Äî after honeypot, before labyrinth (~line 475)
**Config**: New fields in `dashboard/admin.go:flagValues` + admin UI section

#### 1. Graduated Tar Pit
- **File**: `internal/budgettrap/tarpit.go`
- **How**: Track per-client request count via `metrics.Collector.GetClientProfile()`. First 10 reqs: normal. 11-50: mix 20% delays (1-5s). 51-100: 40% slow (10-30s), media links in every page. 100+: majority keepalive abuse, session timeouts, media tar pits
- **Config**: `tarpit_enabled`, `tarpit_threshold` (default 10), `tarpit_max_delay_ms`
- **Wire**: New check in `dispatch()` before error injection

#### 2. Infinite API Pagination
- **File**: `internal/api/pagination_trap.go`
- **How**: Intercept `/api/v1/*/page=N` patterns. Use existing `pathSeed()` + `deterministicUUID/Email/Timestamp` helpers. Set `total: 999999999`, always return `next` link. Generate realistic fake user/product data per page.
- **Config**: `infinite_pagination_enabled`
- **Existing patterns**: Follow `paginatedJSON()` and `PaginationMetadata` from `api/router.go`

#### 3. Fake Vulnerability Breadcrumbs
- **File**: `internal/budgettrap/breadcrumbs.go`
- **How**: Inject headers (`X-Debug-Trace`, `X-Powered-By: PHP/5.4.0`, `Server: Apache/2.2.14`), partial stack traces in HTML comments (`<!-- Error in /var/www/app/db.php line 42 -->`), commented-out SQL in responses, fake `.git/HEAD` refs in meta tags
- **Config**: `breadcrumbs_enabled`
- **Integration**: Called from `servePage()` and error responses

#### 4. Progressive Content Expansion
- **File**: `internal/budgettrap/expansion.go`
- **How**: Each page generates N links where N = `min(baseLinks * depth^1.5, maxLinks)`. Links point to deterministic sub-paths using `pathSeed()`. First level: 5 links. Level 2: ~14 links each. Level 3: ~26 links each. Exponential crawl budget burn.
- **Config**: `progressive_expansion_enabled`, `expansion_base_links` (default 5), `expansion_max_links` (default 50)
- **Integration**: Injected into content engine's `generatePage()`

#### 5. WebSocket Trap
- **File**: `internal/websocket/honeypot.go`
- **How**: New endpoint `/ws/stream` (or enhanced existing). Phase 1: fast messages (build confidence). Phase 2: 30s intervals (fake network degradation). Phase 3: message storm then silence. Never close ‚Äî client must timeout.
- **Config**: `websocket_trap_enabled`
- **Existing patterns**: Follow `wsEndpoint` struct and `connRunner` from `websocket/handler.go`

#### 6. Streaming Response Bait
- **File**: `internal/budgettrap/streaming.go`
- **How**: `Transfer-Encoding: chunked` with first chunk immediate (legit-looking HTML head), then 1 byte/3-10 seconds. Use `http.Hijacker` for connection control. Return HTTP 200, no `Content-Length`.
- **Config**: `streaming_bait_enabled`, `streaming_bait_interval_ms` (default 3000)
- **Integration**: New error type `ErrStreamingBait` in error generator, or standalone handler

## Task 3 ‚Äî MCP Feature: Fake Server + Scanner Support + Admin Integration
**Branch**: `feature/mcp-server`
**Status**: COMPLETE ‚Äî PR #43 merged

### Research Complete ‚Äî Key Findings

**Protocol**: JSON-RPC 2.0 over Streamable HTTP (POST/GET/DELETE single endpoint). SSE for streaming. Session via `Mcp-Session-Id` header.

**Known Vulns (Adversa AI TOP 25 + Prompt Security Top 10)**:
- Tool poisoning (hidden instructions in descriptions)
- Rug pulls (tool behavior changes after approval)
- Cross-server data exfiltration
- Token/credential theft
- Command injection (43% of MCP servers!)
- Path traversal (22%), SSRF (30%)
- No auth by default
- Denial of wallet (budget drain via recursive/slow responses)

**Implementation plan** ‚Äî new `internal/mcp/` package:
```
internal/mcp/
  server.go          ‚Äî HTTP handler (Streamable HTTP + legacy SSE)
  jsonrpc.go         ‚Äî JSON-RPC 2.0 parsing/routing
  session.go         ‚Äî Session management
  tools.go           ‚Äî Tool registry (honeypot + real admin tools)
  resources.go       ‚Äî Resource registry (fake sensitive files)
  prompts.go         ‚Äî Prompt templates (poisoned + legit)
  honeypot.go        ‚Äî Trap logic (credential harvesters, poisoned descriptions)
  fingerprint.go     ‚Äî Agent fingerprinting from initialize clientInfo
```

**Honeypot tool categories**:
1. Credential harvesters (`get_aws_credentials`, `get_api_keys`)
2. Poisoned descriptions (hidden `<IMPORTANT>` injection blocks)
3. Budget drain tools (slow/infinite SSE responses)
4. Data harvesting (accept arbitrary context as params)
5. Misleading data (return plausible but wrong results)

**Admin MCP (behind auth)**:
- Scanner: `start_scan`, `get_results`, `list_profiles`
- Server: `toggle_feature`, `set_error_profile`, `get_metrics`, `nightmare_toggle`
- Proxy: `set_mode`, `set_chaos_probability`

**Scanner MCP module**:
- Connect to target MCP servers
- Analyze tool descriptions for injection
- Test tool calls with payloads
- Test resources for path traversal/SSRF
- Monitor for rug pulls via tool hash comparison
